## ЁЯУВ Project File Structure & Git Submission Guide

When writing your features, **do not write code in raw text files (`.txt`).** All code must be implemented directly in Visual Studio using proper C# files and added to the folder structure below. This keeps our unified namespace compilation working and prevents breaking the `.csproj` file during merges.

```text
ЁЯУБ TetheredGame
  ЁЯУБ Assets                 <-- All .png sprite sheets, audio files, and UI overlays
  ЁЯУБ Core                   <-- Global engine logic, GameLoop, Input managers
  ЁЯУБ Models                 <-- Pure C# classes (.cs files) containing data properties
  ЁЯУБ UI                     <-- WinForms UserControls (.cs, .Designer.cs, .resx files)
  ЁЯУД Program.cs             <-- Application entry point
  ЁЯУД MainGameForm.cs        <-- Main Game Window hosting active UI views

```

### What to commit inside your branches:

1. **If working on files like `feature/engine-grid-movement` or `feature/survival-systems-oxygen`:**
* You will create/modify C# data models inside the `ЁЯУБ Models` directory (e.g., `Player.cs`, `Enemy.cs`, `Tile.cs`).
* Commit actual **`.cs`** files containing pure class objects.


2. **If working on files like `feature/turn-based-combat` or `feature/interactive-puzzles`:**
* You will create modular screens inside the `ЁЯУБ UI` directory using **WinForms UserControls**.
* You must commit three linked files generated by Visual Studio for each visual UI panel:
* `YourControl.cs` тАФ The core C# logic (button click events, calculations).
* `YourControl.Designer.cs` тАФ The system-generated layout definitions.
* `YourControl.resx` тАФ Local interface xml resources.




3. **If working on files like `feature/map-renderer-fog`:**
* Put pixel art textures (`.png` files) into the `ЁЯУБ Assets` directory.
* Put rendering classes into `ЁЯУБ Core` to draw onto the main Graphics canvas.



> ЁЯТб **Visual Studio Integration Rule:** > Make sure you only add and edit these files inside Visual Studio while you are on your own Git branch! **Don't drag and drop files directly into the folders using Windows File Explorer**, or Visual Studio won't register them in the project (`.csproj` configuration), which will break the build for other team members.

## ЁЯУП Strict Coding & Naming Guidelines

Every class, variable, and file in this codebase must adhere to these rules. This ensures our separate scripts plug into each other seamlessly without compilation errors!

### 1. Case Conventions

* **PascalCase** for Class names, Properties, and Method names.
* **camelCase** for local variables and method parameters.
* **UPPERCASE_WITH_UNDERSCORES** for constants.

```csharp
public class PlayerCharacter // PascalCase Class
{
    public const int MAX_OXYGEN = 100; // UPPERCASE Constant

    public int CurrentHP { get; set; } // PascalCase Property
    public bool HasFlashlight { get; set; } // Boolean prefixed with 'Has'

    public void MoveToTile(int targetX, int targetY) // PascalCase Method, camelCase parameters
    {
        bool isWalkable = CheckCollision(targetX, targetY); // camelCase local variable
    }
}

```

Ensure your custom classes are saved in the correct physical folders so namespaces match up:

        // Inventory Systems
        public int InventoryCapacity { get; set; }   // Max storage capacity
